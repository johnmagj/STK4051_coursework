Ch. 9. Bootstrapping
- Direction of Causality: Ex. weight and hight can mathematically both be functions of each other, but biologically hight is more "set" while weight can fluctuate, therefore more "natural" with weight as a function of hight: weight(hight)

- Non-parametric: re-sample from the original data sample
- Parametric: re-sample from a distribution (with estimated parameters) we assume represent the population that the original sample came from

- Important example of usage: calculate confidence intervals directly (from percentiles), find bias in the our estimates and correct the estimates for the bias, find standard error for the corrected estimator

- Bootstrapping basic:
	- In bootstrapping we assume the sample is representative of the population, i.e. it came from a population with an underlaying distribution (but with some possible bias)
	- GOOD for estimating the spread (variance) in the population and the population distribution, e.g. the spread in the sample can be used to say something about the spread in the population
	- BUT can not correct for a bias mean, i.e. if the mean of the sample is 50, but really 60 for the population, we wont catch that, ONLY the spread around the mean

- Main ways to conduct bootstrapping:
	- Percentile method:
		1. You calculate some estimate from the original sample
		2. Sample from original sample with replacement
			2.1. Calculate bootstrap estimate
			2.2. Store bootstrap estimate
		3. Use the stored bootstrap estimates to create histograms or calculate confidence intervals

	- Pivot/Studentized:
		1. From the original sample calculate estimate thetha_hat
		2. For each bootstrap calculate some "distance" or "t-score", like (theta_hat_star - theta_hat)/SE_star, where SE_star is the standard error of the bootstrap sample.

- Bootstrapping workflow:
	- 1. Do 1000 bootstraps of original sample (with replacement)
	- 2. Estimate on the original sample, i.e. find and estimate using the original sample, the value is a point estimate
	- 3. Estimate on each bootstrap sample, called bootstrap estimates
	- 4. Histogram of bootstrap estimates
	- 5. Find the mean of the bootstrap estimates
	- 6. Calculate confidence interval for the estimate, directly using percentiles or with BCa (for highly skewed data)
	- 7. Find bias for estimate with: bias = mean_of_bootstrap_estimates - original_estimate
	- 8. Find bias corrected estimate with: bias_corrected_estimate = original_estimate - bias
	- 9. Calculate the standard error, for bootstraps simply the std of the estimate resulted from each bootstrap sample

- Bootstrapping: allows for simulation of the distribution of a statistic (like mean, standard deviation, median)
		- Can be used both for i.i.d. data and and dependant data (but beneficial to consider the stochastic mechanism generating the data and the resampling strategy)
		- With the aim to "explorer" one statistic, e.g. the mean

- BCa (Accelerated Bias-Corrected Percentile Method):
	- Bootstrap method used to create more accurate confidence intervals the standard percentile method, by adjusting bias and skewedness (acceleration)
	- Bias correction: 
	- Acceleration(skewedness)
	- For non symmetric (skewed) data, or original sample statistic is biased
	- Provide second or accuracy --> error rate shrink faster as sample size increases compared to standard method

- Bootstrapping and regression:

	- Residual bootstrapping: 
		1. fit model on the original sample of X and y
		2. predict on X and use the prediction to find the residuals, y - pred
		2.5. (If no intercept, else optional) center the residuals by subtracting the residual mean
		3. Bootstrap from the residuals (with replacement)
		4. Add the residuals to the original prediction, y* = residuals* + prediction
		5. fit new model on X and y*, store the parameters per bootstrap

		- like strict teacher, only noise can move, x values are fixed/frozen (we draw residuals not x values)
		- assume linear relationship

	- Pair (cases) bootstrapping:
		1. Pair up the data points (x, y)
		2. Draw bootstrap samples from the set of pairs
		3. For each bootstrap sample fit model to X* and y*, store the parameters
	
		- like relaxed teacher, the pair (x,y) can move/drawn randomly
		- assume nothing about the underlaying model, treat the pairs completely randomly
		- stricter, more conservative

- The Empirical distribution function (EDF): The function representing a sample, reflects the proportions in the sample i.e. if the sample consists of 5 flips of a coin and there are 3 Hs and 2 Ts then sampling from this distribution would yield a higher probability for Heads, the function we sample from when creating a bootstrap sample
	- Considered non parametric since we make no assumptions about the distribution paramters, it is all there in the sample
- for empirical distribution, no need for z- or t-score as in normal confidence interval (which relay on theoretical distribution like the normal distribution)

- Hypothesis testing: With an estimate theta_hat from our original sample, and a null value theta_0 (null hypothesis):
	1. Bootstrap from original sample with replacement
		1.1. For each bootstrap sample calculate a bootstrap estimate theta_hat_star
		1.2. Calculate: (theta_hat_star - theta_hat), i.e. the bootstrap estimate minus the original estimate
		1.3. Store the result
	2. Use the calculated bootstrap values and compare with (theta_hat - theta_null) (difference between original theta and the null hypothesis). How many of the bootstrap values are greater then original value theta_hat - theta_null?

	

- Balanced (permuted) bootstrap: 
	1. Make B copies of the data (size n)
	2. Concatenate (putting them after one another) all these samples
	3. Randomly permute this series of values
	4. Read of B blocks of size n sequentially.

- Antithetic bootstrap:
	1. Calculate some statistic on the original dataset
	2. Sort the result
	3. Resample from the sorted result list, for each draw make pick also the corresponding result starting from the oposite end of the list, thus we resample pairs across the center of the sorted list

- Bootstrapping dependant data, for instance timeseries data where subsequent datapoints are correlated:
	- Model based approach:
 
	- Block Bootstrap:
		- Non-moving block bootstrap: 
			1. Data, say: [1,2,3,4,5,6,7,8,9] is split into blocks, e.g. [1,2,3], [4,5,6], [7,8,9]
			2. Resample the blocks with replacement, for instance we can get: [4,5,6], [1,2,3], [1,2,3]
			- Depending on block size, can miss trends in data. Considered inferior to Moving block bootstrap

		- Moving block bootstrap:
			l = block size (number of individual samples per block)
			n = number of individual samples in original dataset
			b = number of blocks
			l-1 = overlap
			n-l+1 = number of blocks = b
			1. Data, say: [1,2,3,4,5,6,7,8,9] is split into overlapping blocks, e.g [1,2,3], [2,3,4], [3,4,5],
[4,5,6], [5,6,7], [6,7,8], [7,8,9]
			2. Resample these blocks with replacement, if bootstrap sample is bigger then original sample, truncate

		- Blocks-of-Blocks bootstrapping: 
			- Used to check accuracy of first bootstrap, since choosing block length is a guess
			1. Perform a moving block bootstrap
			2. For each bootstrap resample, resample from this with another moving block bootstrap
			3. Comparing the two bootstrap samples, two see how much they vary. The error is used to calibrate/adjust the original bootstrap

		- Block size:
			- block size, l, l=1 correspond to standard bootstrap. Big l will represent larger trends in the data but leave fewer blocks to resample from
			- block length should increase with dataset size
